Pathfinders engineering · how we build with AI

Claude burns the tokens; the team collects the XP

We run Claude Code and GitHub Copilot as an assembly line, not an oracle. Coordinated AI sessions do the volume; review gates and human judgment do the tempering. Tokens are a cost we spend and forget — the craft they buy stays on the balance sheet.

The loop

The distinction that matters

Why this isn't vibe-coding

Vibe-coding is trusting the machine on a hunch and shipping what looks right. Same tools, opposite discipline — the gates are the whole difference.

Vibe-coding

spends heat, keeps nothing

  • Accept the model's output on vibes; ship what reads plausibly
  • No evidence, no citation — plausibility passes for proof
  • No review gate; the human is a rubber stamp
  • Fast today, a mystery at 3am — nothing was learned

How Pathfinders runs it

tempers heat into an edge

  • Evidence before output — every claim traces to code, a log, a query
  • Copilot reviews every draft PR before a human reads it
  • The engineer owns and can overrule the model — never self-merge
  • Tests prove it; code-reviewers gate main. Speed stays honest

The balance sheet

What compounds

Tokens are a renewable cost — burned and gone. These four are what the spend actually buys, and they don't reset next sprint.

Judgment

Engineers read, verify, and overrule the model. The skill that grows is knowing when it's wrong — the one thing a tool can't hand you.

Skills

A personal Claude skill that proves itself graduates into the shared ezrd team plugin — one engineer's shortcut becomes everyone's.

Doctrine

Hard-won gotchas land in CLAUDE.md. A trap that burned us once is a guardrail forever — nobody relearns it the hard way.

Runbooks

The wiki captures the path once it's walked, so the next engineer covers in an hour what took the first one a day.

Tokens are rented. Craft is owned.

The loop closes
